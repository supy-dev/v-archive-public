# Feature Specification: 神回登録・神回お気に入りページ改修・タイムスタンプメモ保管庫の新設

**Feature Branch**: `007-legendary-and-favorites`

**Created**: 2026-06-22

**Status**: Draft

**Input**: User description: "現在の「タイムスタンプメモ」と「神回・お気に入り」画面の責務を整理しました。こちらを元に、改善対応するためのスペックを作成してください。また、動画を「神回に登録」（user_watch_items.is_favorite = true）の実装も実施したいです。"

## 情報設計の前提

| ページ | URL | 役割 | サイドバー |
|--------|-----|------|-----------|
| 神回・お気に入り | `/favorites` | 厳選コレクション（神回動画タブ ＋ ★メモタブ） | 「神回・お気に入り」 |
| タイムスタンプメモ | `/memos` | 保管庫（★問わず全メモ、検索・絞り込み） | 「タイムスタンプメモ」 |

両ページはサイドバーで独立した導線を持ち、統合しない。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 動画を神回に登録する (Priority: P1)

配信詳細ページを閲覧中のユーザーが、その動画を「神回」として登録・解除できる。登録すると `user_watch_items.is_favorite = true` が保存され、神回・お気に入りページの「神回」タブに現れる。

**Why this priority**: `user_watch_items.is_favorite` カラムは004から存在するが操作するUIがない。これが本機能追加の核心であり、US2（神回タブ表示）の前提となる。

**Independent Test**: 配信詳細ページで「神回に登録」ボタンを押すと、ページ遷移なしにボタンが登録済み状態になる。その後「神回・お気に入り」ページの「神回」タブを開くと当該動画が表示される。

**Acceptance Scenarios**:

1. **Given** 動画が未登録の状態で配信詳細ページを開いているとき、**When** 「神回に登録」ボタンを押すと、**Then** ボタンが登録済み状態に変わり、サーバーに `is_favorite = true` が保存される。
2. **Given** 動画が神回登録済みのとき、**When** 「神回解除」ボタンを押すと、**Then** ボタンが未登録状態に戻り、サーバーに `is_favorite = false` が保存される。
3. **Given** サーバーエラーが発生したとき、**When** トグル操作を行うと、**Then** エラーメッセージが表示され、ボタン状態は変化しない。

---

### User Story 2 - 神回・お気に入りページを2タブ構成に刷新する (Priority: P2)

「神回・お気に入り」ページ（`/favorites`）が「神回」と「お気に入りメモ」の2タブで厳選コレクションを管理できるよう刷新する。現状は★メモのみ表示しているが、神回動画タブを追加する。

**Why this priority**: US1 で神回登録できるようになっても、一覧側に動画タブがなければ価値が完結しない。P1 の直後に実装する。

**Independent Test**: 神回動画を1件登録した状態で「神回・お気に入り」ページを開き、「神回」タブに動画カードが表示される。「お気に入りメモ」タブには従来通り★メモが表示される。

**Acceptance Scenarios**:

1. **Given** 神回動画が1件以上あるとき、**When** 「神回」タブを開くと、**Then** 動画タイトル・サムネイル・チャンネル名・登録日が表示され、配信詳細へのリンクがある。
2. **Given** 神回動画が0件のとき、**When** 「神回」タブを開くと、**Then** 神回登録を促す空状態メッセージが表示される。
3. **Given** 「お気に入りメモ」タブを開くとき、**When** タブを切り替えると、**Then** `timestamp_memos.is_favorite = true` のメモ一覧が表示される（既存の `/favorites` 機能を維持）。
4. **Given** 推し別・年月別フィルタを適用したとき、**When** フィルタを変更すると、**Then** 現在表示中のタブに対して絞り込みが機能する。
5. **Given** いずれかのタブを開いているとき、**When** URL を確認すると、**Then** アクティブなタブが URL パラメータ（例: `?tab=favorites`）に反映されており、ブックマーク可能である。

---

### User Story 3 - タイムスタンプメモ保管庫 `/memos` を新設し、サイドバー導線を接続する (Priority: P3)

「書き留めたもの全部」を探せる `/memos` 画面を新規実装し、現在ホームの自己アンカーに戻るだけのサイドバー「タイムスタンプメモ」導線・ホームの「もっと見る」を正しく接続する。

**Why this priority**: サイドバーに「タイムスタンプメモ」の項目があるにもかかわらず、対応する画面が存在しない。P1/P2 完了後に取り組む。

**Independent Test**: サイドバーの「タイムスタンプメモ」を押すと `/memos` へ遷移し、★の有無を問わず自分の全タイムスタンプメモが表示される。推し・タグ・年月で絞り込める。

**Acceptance Scenarios**:

1. **Given** メモが1件以上あるとき、**When** `/memos` を開くと、**Then** ★の有無を問わず全タイムスタンプメモが新しい順に表示される。
2. **Given** 推し・タグ・年月フィルタを適用したとき、**When** フィルタを変更すると、**Then** 一覧が絞り込まれ、フィルタ値が URL に保持される。
3. **Given** 任意のページでサイドバーを表示しているとき、**When** 「タイムスタンプメモ」を押すと、**Then** `/memos` へ遷移し、ナビ項目が選択状態になる。
4. **Given** ホームの「最近のタイムスタンプ」セクションを表示しているとき、**When** 「もっと見る」を押すと、**Then** `/memos` へ遷移する。
5. **Given** ホームの「最近のタイムスタンプ」が固定文言のとき、**When** 実データに差し替えると、**Then** ログインユーザーの最新3件のタイムスタンプメモが表示される。
6. **Given** `/memos` の各メモカードを確認したとき、**When** 主リンクを押すと、**Then** アプリ内の配信詳細ページへ遷移する（YouTube への直接遷移ではない）。
7. **Given** `/memos` のメモカードを確認したとき、**When** ★ボタンの存在を確認すると、**Then** ★トグルボタンは表示されない（配信詳細ページ専用の操作）。

---

### Edge Cases

- 動画が削除済み（`is_available = false`）の場合でも、神回登録ボタンは表示・操作できる。
- 同時に複数タブで神回トグルを操作した場合、後から送信されたリクエストが最終状態となる（last-write-wins）。
- 神回動画タブでフィルタを適用した結果0件になった場合、タブに応じた「該当なし」メッセージを表示する。
- `/memos` でメモが0件のとき（新規ユーザー等）は、メモ追加を促す空状態メッセージを表示する。
- ホームの「最近のタイムスタンプ」がまだ0件の場合は、空状態を表示し、固定文言ではなく実データ件数（0件）を反映する。
- 感想メモ（`video_notes`）は `/memos` 保管庫に含めない（配信詳細専用の責務）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは配信詳細ページに「神回に登録／解除」トグルボタンを提供しなければならない（MUST）。
- **FR-002**: トグルボタンを押すと、ページ遷移なしに `user_watch_items.is_favorite` がサーバー側で更新されなければならない（MUST）。
- **FR-003**: トグルボタンは `user_watch_items.is_favorite` の現在値を反映した初期状態で描画されなければならない（MUST）。
- **FR-004**: 神回トグルのエンドポイントは所有権チェック（Policy）を通過したユーザーのみ更新を許可しなければならない（MUST）。
- **FR-005**: `/favorites` ページは「神回」タブと「お気に入りメモ」タブの2タブ構成に刷新されなければならない（MUST）。
- **FR-006**: 「神回」タブは `user_watch_items.is_favorite = true` の動画を、動画タイトル・サムネイル・チャンネル名・登録日付きで表示しなければならない（MUST）。
- **FR-007**: 「神回」タブの各動画カードはアプリ内の配信詳細ページへのリンクを持たなければならない（MUST）。
- **FR-008**: 「お気に入りメモ」タブは現行の `/favorites` 実装（`timestamp_memos.is_favorite = true` のフィルタ・ページネーション）を維持しなければならない（MUST）。
- **FR-009**: `/favorites` のフィルタ（推し別・年月別）は現在アクティブなタブに対して機能しなければならない（MUST）。
- **FR-010**: アクティブなタブは URL パラメータに反映され、ブックマーク・ブラウザ履歴で再現できなければならない（MUST）。
- **FR-011**: システムは `/memos` ルートを新設し、ログインユーザーの全タイムスタンプメモを表示しなければならない（MUST）。
- **FR-012**: `/memos` は推し別・タグ別・年月別フィルタを提供しなければならない（MUST）。
- **FR-013**: `/memos` の各メモカードはタイムスタンプ・本文・タグ・動画タイトル・チャンネル名・作成日を表示し、アプリ内配信詳細を主リンクとしなければならない（MUST）。
- **FR-017**: `/memos` はお気に入り（★）トグル操作を提供しない。★の付け外しは配信詳細ページのみで行う（MUST NOT）。
- **FR-014**: サイドバーの「タイムスタンプメモ」ナビゲーション項目は `/memos` へリンクしなければならない（MUST）。
- **FR-015**: ホームの「最近のタイムスタンプ」の「もっと見る」は `/memos` へリンクしなければならない（MUST）。
- **FR-016**: ホームの「最近のタイムスタンプ」はログインユーザーの実データ（最新3件）を表示しなければならない（MUST）。

### Key Entities

- **UserWatchItem**: ユーザーと動画の紐付けレコード。`is_favorite` フィールドで神回フラグを管理。既存カラム、マイグレーション不要。
- **TimestampMemo**: タイムスタンプメモ。`is_favorite` フィールドで★フラグを管理。既存カラム、マイグレーション不要。
- **YoutubeVideo**: 共有マスタ動画情報（タイトル・サムネイル・duration）。読み取り専用。
- **YoutubeChannel**: 共有マスタチャンネル情報（チャンネル名・アバター）。読み取り専用。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 配信詳細ページで「神回に登録」ボタンを押してから状態が切り替わるまでの体感遅延が 1 秒以内である。
- **SC-002**: 「神回・お気に入り」ページを開いたとき、神回動画とお気に入りメモがそれぞれのタブに正しく分離表示される。
- **SC-003**: `/memos` を開いたとき、★の有無を問わず全タイムスタンプメモが一覧表示される。
- **SC-004**: Desktop（960px）・Mobile（390px）の両サイズで、横スクロールなしにすべての操作要素が正しく表示される。
- **SC-005**: サイドバーの「神回・お気に入り」「タイムスタンプメモ」がそれぞれ意図した画面へ正しく遷移する。
- **SC-006**: ホームの「最近のタイムスタンプ」が実データを表示し、「もっと見る」が `/memos` へ遷移する。
- **SC-007**: 既存のタイムスタンプメモ★機能・感想メモ機能・配信詳細プレイヤーが本機能追加後も正常に動作する（リグレッションなし）。

## Clarifications

### Session 2026-06-22

- Q: `/memos` でお気に入り（★）トグルを操作できるか？ → A: できない。★トグルは配信詳細ページのみ。`/memos` は閲覧・絞り込み専用。
- Q: `/favorites` を開いたときのデフォルトタブはどちらか？ → A: 「神回」タブをデフォルトにする。

## Assumptions

- `user_watch_items.is_favorite` および `timestamp_memos.is_favorite` カラムはすでに存在する（マイグレーション不要）。
- 神回動画の登録操作は配信詳細ページのみに UI を追加する（アーカイブ一覧からの一括操作は対象外）。
- `/memos` はタイムスタンプメモ（`timestamp_memos`）専用の保管庫であり、感想メモ（`video_notes`）は含めない。
- 神回動画タブのソート順は「登録日（`user_watch_items.updated_at`）降順」を基本とする。
- `/memos` のソート順は「作成日 降順」を初期値とする。
- 神回動画タブのフィルタは推し別・年月別とする（動画自体にタグが紐づかないため、タグフィルタは対象外）。
- 既存の `TimestampMemoPolicy` と `UserWatchItemPolicy` の所有権チェックをそのまま活用する。
- ホームの「最近のタイムスタンプ」実データ化は既存の `HomeController` のクエリ変更で対応する（新Controller不要）。
- ホームの「神回リスト」バナーのリンク先は `/favorites` のまま維持する（デフォルトタブが神回のため、タブ指定パラメータは不要）。
- `/favorites` をパラメータなしで開いた場合は「神回」タブをデフォルト表示する。URLパラメータ（例: `?tab=memos`）で「お気に入りメモ」タブへの直接遷移も可能とする。
- UI 変更は CLAUDE.md のデザインガイドラインに準拠し、新しいカラーシステムやフォントスタックは導入しない。
