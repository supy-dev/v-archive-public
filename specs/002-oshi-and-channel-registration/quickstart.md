# Quickstart Validation Guide: Oshi & Channel Registration

本ガイドは Feature 002 が仕様どおりに動作することを確認するための**検証シナリオ集**。
実装コードや完全なテストスイートは `tasks.md` と実装フェーズで扱う。

## 前提条件

- Feature 001（認証基盤）が完成しており、ユーザーがログインできる状態。
- YouTube API キーが `.env` に設定済み（`YOUTUBE_API_KEY=...`）。
- Docker Compose が起動済み（`docker compose up -d`）。
- マイグレーション適用済み（`php artisan migrate`）。

## 検証の実行

```bash
# Feature/Unit テスト一括実行（YouTube API は Http::fake でモック済み）
php artisan test --filter="Oshi|Channel|YouTube"

# または全テスト
php artisan test
```

## シナリオ 1: 推し作成から一覧表示まで（US1 P1）

**目的**: FR-001 / FR-002 / SC-001 の基本フローを確認。

### 1-A. 推しを作成する

1. ログイン済みで `/oshis/create` を開く。
2. **name** に「姫森ルーナ」、**group_name** に「ホロライブ」、**color_id** でパレットからピンクを選択。
3. 送信する。

**期待**: `/oshis/{id}` へリダイレクト。推し詳細ページに名前・グループ・テーマカラーが表示される。

### 1-B. 推し一覧に表示される

1. `/oshis` を開く。

**期待**: 作成した推しが一覧に表示される（チャンネル登録数: 0）。

### 1-C. 名前なし作成は拒否される

1. `/oshis/create` で name を空にして送信。

**期待**: `422`。フォームに「名前は必須です」相当のエラーが表示。推しは作成されない。

### 1-D. パレット外のカラーは拒否される

1. API 経由で `color_id=invalid_color` を POST。

**期待**: `422`。推しは作成されない。

### 1-E. 推しを編集する

1. `/oshis/{id}/edit` を開き、group_name と memo を変更して送信。

**期待**: 詳細ページに変更が反映されている。

### 1-F. 推しを削除する

1. `/oshis/{id}` で削除操作を行う。

**期待**: 一覧から消える。後述の紐づくチャンネル登録も削除されていることを DB で確認。

## シナリオ 2: YouTube チャンネル登録（US2 P1）

**目的**: FR-003 / FR-004 / FR-005 / FR-006 / FR-007 / SC-002 / SC-003 / SC-004 を確認。

### 2-A. チャンネル URL で登録する

1. 推し詳細（`/oshis/{id}`）を開く。
2. チャンネル登録フォームに `https://www.youtube.com/channel/UCQ0e7K7IPBFV_WWWFZ1VEgg` を入力して送信。

**期待**: チャンネルが登録一覧に表示される（タイトル・サムネイル・「同期待ち」状態）。
`youtube_channels` に1件レコードが作成される。`user_channels.is_main = true`（初回なので）。

### 2-B. @handle で登録する

1. 別の推しの詳細を開く。
2. `@hoshimachi_suisei` を入力して送信。

**期待**: チャンネルが特定されて登録される。

### 2-C. handle URL で登録する

1. `https://www.youtube.com/c/fubukisch` を入力して送信。

**期待**: チャンネルが特定されて登録される。

### 2-D. 対応形式外は拒否される

1. `https://www.google.com/` を入力して送信。

**期待**: `422`。「対応形式: ...」のメッセージが表示される。チャンネルは登録されない。

### 2-E. 存在しないチャンネルは拒否される

1. `@a_channel_that_does_not_exist_12345` を入力して送信
   （テストでは `Http::fake` で空レスポンスを返す）。

**期待**: `422`。「チャンネルが見つかりませんでした」相当のメッセージ。

### 2-F. 重複登録は防止される

1. シナリオ 2-A で登録済みのチャンネルを同じ推しに再度登録しようとする。

**期待**: `422`。「このチャンネルはすでに登録されています」メッセージ。`user_channels` は増えない。

### 2-G. 別ユーザーが同一チャンネルを登録する（共有マスタ再利用）

1. 別のユーザーアカウントでログインし、2-A と同じチャンネル URL を登録する
   （テストでは 2 ユーザーを `actingAs` で切り替え）。

**期待**: `youtube_channels` に新規レコードは作成されない（1件のまま）。
`user_channels` に別ユーザーの登録が1件追加される。

### 2-H. YouTube API エラー時でも画面が停止しない

テスト: `Http::fake` で `500` を返した場合。

**期待**: `422`。「しばらく後に再度お試しください」相当のメッセージ。スタックトレースは表示されない。

## シナリオ 3: 推しとチャンネルの一覧・確認（US3 P2）

**目的**: FR-002 / SC-007（レスポンシブ）を確認。

### 3-A. 推し詳細にチャンネルが表示される

1. 推しに2つのチャンネルを登録した状態で詳細ページを開く。

**期待**: チャンネルが一覧表示される（タイトル・サムネイル・同期状態「同期待ち」・メイン指定の視覚的な区別）。

### 3-B. 別ユーザーの推し詳細 URL へ直接アクセス

1. 他ユーザーが持つ推しの URL（`/oshis/{他ユーザーの推しid}`）を直接開く。

**期待**: `403 Forbidden`。

## シナリオ 4: チャンネル設定とメインチャンネル管理（US4 P2）

**目的**: FR-009 / FR-012 / SC-008 / FR-017 を確認。

### 4-A. 最初のチャンネルは自動的にメインになる

1. 推しへ初めてチャンネルを登録する。

**期待**: `user_channels.is_main = true`。詳細画面でメインチャンネルとして表示される。

### 4-B. 2件目以降の登録はメインにならない

1. 同じ推しに2件目のチャンネルを登録する。

**期待**: 2件目の `is_main = false`。メインは1件目のまま。

### 4-C. メインチャンネルを変更する

1. 推し詳細で2件目をメインに指定する。

**期待**: 1件目の `is_main = false`。2件目の `is_main = true`。常にちょうど1つ。

### 4-D. メインチャンネルの登録を解除すると次がメインになる

1. メインチャンネルの登録を解除する（他にチャンネルがある状態）。

**期待**: 残ったチャンネルの中で最古の `registered_at` を持つチャンネルが自動的にメインへ昇格する。
SC-008「メインは常にちょうど1つ」を維持。

### 4-E. 同期を無効にする

1. チャンネル設定で `sync_enabled` を OFF にする。

**期待**: 設定が保存される（`user_channels.sync_enabled = false`）。

### 4-F. 通知設定を変更する

1. `notify_enabled` を ON にする。

**期待**: 設定が保存される（`user_channels.notify_enabled = true`）。
実際の通知配信は行われない（本フィーチャーの範囲外）。

### 4-G. チャンネル登録を解除しても共有マスタは残る

1. `user_channels` の登録を解除する。

**期待**: 自分の `user_channels` レコードのみ削除。`youtube_channels` の共有マスタは残る。
他ユーザーがそのチャンネルを登録している場合も影響なし。

## セキュリティ確認チェックリスト

- [ ] 未認証でアクセスすると `/login` へリダイレクトされる（全 `/oshis/**` ルート）。
- [ ] 他ユーザーの推しの詳細・編集・削除が `403` になる。
- [ ] 他ユーザーのチャンネル登録の更新・解除が `403` になる。
- [ ] YouTube API キーがレスポンス HTML・JavaScript に含まれない。
- [ ] API エラー時の内部スタックトレースがユーザーに表示されない。
- [ ] チャンネル登録フォームの CSRF トークンが機能する。
- [ ] レート制限: 短時間に大量の POST を送ると `429` が返る。
