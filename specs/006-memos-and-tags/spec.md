# Feature Specification: メモ・タグ・神回お気に入り

**Feature Branch**: `006-memos-and-tags`

**Created**: 2026-06-20

**Status**: Draft

**Input**: User description: "feature 6の仕様を作成してください！"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タイムスタンプメモの記録とシーク (Priority: P1)

動画を視聴しながら「今の場面が面白かった」「この歌が良かった」と感じた瞬間に、
現在の再生位置をメモとして保存できる。保存したメモ一覧から任意のタイムスタンプを
タップ/クリックすると、その時刻から動画が再生される。

**Why this priority**: メモ機能はこのFeatureの中核価値であり、視聴体験に直結する。
タイムスタンプ保存とシークがないと他のメモ機能もテストできない。

**Independent Test**: 動画詳細画面で「現在位置をメモ」ボタンを押し、本文を入力して保存する。
保存済みメモ一覧にタイムスタンプと本文が表示され、タイムスタンプをクリックすると
YouTube IFrame Playerが該当位置から再生されることを確認する。

**Acceptance Scenarios**:

1. **Given** 動画を再生中（例: 再生位置3分20秒）、**When** 「現在位置をメモ」ボタンを押す、**Then** 入力欄が開き初期値として「3分15秒」（現在位置から5秒前）が設定される
2. **Given** タイムスタンプと本文を入力して保存した、**When** メモ一覧を表示する、**Then** タイムスタンプ（例: 3:15）と本文が一覧に表示される
3. **Given** メモ一覧に保存済みのタイムスタンプが表示されている、**When** タイムスタンプをクリックする、**Then** YouTube Playerが指定秒数から再生を開始する
4. **Given** 認証済みユーザーとして動画詳細を閲覧、**When** 本文を空のままメモを保存しようとする、**Then** バリデーションエラーが表示され保存されない
5. **Given** 別のユーザーのタイムスタンプメモが存在する、**When** そのメモのAPIエンドポイントへ直接リクエストする、**Then** 403 Forbiddenが返る

---

### User Story 2 - 動画全体ノート（全体感想）の保存 (Priority: P2)

動画を見終えた後や途中で、動画全体への感想・メモを自由記述で保存できる。
1動画1ユーザー1件のノートとして管理される。

**Why this priority**: 全体感想はタイムスタンプメモと並ぶ主要メモ機能。配信詳細画面の
下部に配置されており、シンプルなCRUDで独立してテストできる。

**Independent Test**: 動画詳細画面の「全体感想」欄にテキストを入力して保存する。
ページをリロードしても入力内容が維持されていることを確認する。

**Acceptance Scenarios**:

1. **Given** 動画詳細画面を初回表示、**When** 「全体感想」欄を確認する、**Then** 空の入力欄が表示される
2. **Given** 全体感想欄に本文を入力して保存ボタンを押す、**When** ページをリロードする、**Then** 入力した本文が表示される（上書き保存）
3. **Given** 保存済みの全体感想がある、**When** 本文を変更して保存する、**Then** 新しい内容に更新される
4. **Given** 保存済みの全体感想がある、**When** 本文を空にする、**Then** 保存ボタンが無効化され押せなくなる
5. **Given** 保存済みの全体感想がある、**When** 「削除」ボタンを押す、**Then** ノートが削除され、空の入力欄が表示される
5. **Given** 別のユーザーの動画ノートが存在する、**When** そのノートのAPIエンドポイントへ直接更新リクエストする、**Then** 403 Forbiddenが返る

---

### User Story 3 - タグ付与とシステムタグ (Priority: P3)

タイムスタンプメモに「笑った」「泣いた」「歌」「神シーン」などのタグを付与できる。
システムが用意した定型タグと、ユーザーが独自に作成したタグの両方を使用できる。

**Why this priority**: タグはメモの検索・整理を強化する機能だが、タグなしでも
メモ機能は成立するためP3とする。

**Independent Test**: タイムスタンプメモ保存時にシステムタグ「笑った」を選択して保存する。
メモ一覧でタグが表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** タイムスタンプメモ入力画面を表示、**When** タグ選択欄を確認する、**Then** 「笑った」「泣いた」「歌」「てぇてぇ」「重大発表」「神シーン」等のシステムタグが選択肢として表示される
2. **Given** システムタグを1つ以上選択してメモを保存した、**When** メモ一覧を確認する、**Then** 付与したタグが各メモに表示される
3. **Given** タイムスタンプメモのフォームで既存にないタグ名を入力してメモを保存、**When** メモ一覧を確認する、**Then** インラインで作成されたユーザー固有タグがメモに表示される
4. **Given** 別のユーザーのシステムタグが存在する、**When** タグ一覧を参照する、**Then** システムタグ（共有）はすべてのユーザーに表示され、他ユーザーの固有タグは表示されない

---

### User Story 4 - お気に入り登録と神回・お気に入り一覧 (Priority: P4)

気に入ったタイムスタンプメモを「お気に入り」として印を付けられる。
神回・お気に入り一覧画面では、お気に入りのタイムスタンプと動画を一覧表示し、
推し別・年月別・タグ別で絞り込める。

**Why this priority**: お気に入り一覧は独立した画面として提供されるが、
タイムスタンプメモが存在することが前提のためP4とする。

**Independent Test**: タイムスタンプメモの「お気に入り」ボタンを押し、神回・お気に入り
一覧ページへ遷移してそのメモが表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** 保存済みのタイムスタンプメモがある、**When** メモのお気に入りボタンを押す、**Then** ボタンの状態がお気に入り済みに切り替わる
2. **Given** お気に入り登録済みのメモがある、**When** 神回・お気に入り一覧画面を開く、**Then** お気に入りメモのタイムスタンプ・本文・動画タイトルが表示される
3. **Given** 神回・お気に入り一覧を表示中、**When** 推しでフィルタリングする、**Then** 選択した推しの動画に属するお気に入りのみ表示される
4. **Given** お気に入り済みのメモがある、**When** 再度お気に入りボタンを押す（トグル解除）、**Then** お気に入り一覧からそのメモが消える

---

### Edge Cases

- 動画が非公開・削除済み（`is_available = false`）の場合でも保存済みメモは参照・編集できる
- 動画の再生時間が取得できない状態で「現在位置をメモ」を押した場合は0秒を初期値とする
- `seconds = 0`（動画開始直後）のタイムスタンプメモは有効とする
- メモ本文に含まれるHTMLタグ・スクリプトはエスケープして保存・表示する（XSS防止）
- 同一秒数に複数のタイムスタンプメモを保存することは許可する
- タグ名が重複する場合（同一ユーザー固有 or システムタグと同名）の扱いはUNIQUE制約で防ぐ

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは動画の任意の再生位置（秒）と本文を紐付けてタイムスタンプメモを保存できる（MUST）
- **FR-002**: タイムスタンプメモの初期秒数は「現在の再生位置から5秒前」を自動入力する（MUST）
- **FR-003**: 動画詳細画面でタイムスタンプ一覧を表示し、クリックすると対応秒数からYouTube Playerが再生を開始する（MUST）
- **FR-003a**: タイムスタンプメモの保存はサーバーへのリクエスト完了後にレスポンスを受け取り、成功した場合のみ Alpine.js でメモ一覧を更新する。失敗時はエラーメッセージを表示する（MUST）
- **FR-004**: タイムスタンプメモは編集・削除できる（MUST）
- **FR-005**: ユーザーは動画全体の感想（video_note）を1動画1件保存できる。明示的な「保存」ボタンを押すことで保存・上書きし、保存完了後に「保存しました」等のフィードバックを表示する（MUST）
- **FR-006**: 動画ノートの保存ボタンは本文が空のとき無効化する。保存済みノートを削除したい場合は明示的な「削除」ボタンから行う（空ノートはDBに残さない）（MUST）
- **FR-007**: タイムスタンプメモに1件以上のタグを付与・解除できる（MUST）
- **FR-008**: システムタグ（`is_system = true`）は全ユーザーに共有される定型タグとして提供される（MUST）
- **FR-009**: ユーザーはタイムスタンプメモの入力フォーム内でタグ名を直接入力し、ユーザー固有タグをインラインで新規作成できる（MUST）
- **FR-010**: タイムスタンプメモを「お気に入り」として登録・解除できる（`is_favorite`のトグル）（MUST）
- **FR-011**: 神回・お気に入り一覧画面でお気に入り登録済みのタイムスタンプメモを一覧表示する（MUST）。お気に入り動画タブは将来拡張とし本Featureのスコープ外とする
- **FR-012**: 神回・お気に入り一覧は推し別・タグ別・年月別で絞り込みできる（MUST）
- **FR-013**: タイムスタンプメモ・動画ノートへの操作はPolicyで所有権を必ず確認し、他ユーザーのデータを操作できない（MUST）
- **FR-014**: タイムスタンプメモ本文は必須入力とし、最大1000文字以内とする（MUST）
- **FR-015**: 動画ノートの本文は最大5000文字以内とする（MUST）
- **FR-016**: 動画詳細画面の「YouTubeで開く（タイムスタンプ付き）」リンクで、アプリ内タイムスタンプに対応するYouTube URLを生成できる（MUST）
- **FR-017**: 非公開・削除済み動画でも保存済みタイムスタンプメモ・動画ノートは参照・編集できる（MUST）

### Key Entities *(include if feature involves data)*

- **タイムスタンプメモ（TimestampMemo）**: ユーザーが特定の動画の特定秒数（seconds）に紐付けて保存する短文メモ。お気に入りフラグ（is_favorite）を持つ。1ユーザー・1動画に複数件保存可能。
- **動画ノート（VideoNote）**: ユーザーが動画全体に対して記述する自由記述メモ。1ユーザー・1動画につき1件のみ。
- **タグ（Tag）**: タイムスタンプメモに付与できるラベル。システムが提供する共有タグ（is_system=true）とユーザー固有タグの2種類がある。
- **タイムスタンプメモタグ（TimestampMemoTag）**: タイムスタンプメモとタグの多対多中間テーブル。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが動画視聴中に「現在位置をメモ」ボタンを押してからメモ保存完了まで5秒以内に完了できる
- **SC-002**: 保存済みタイムスタンプをクリックしてから動画再生が該当位置から開始されるまで2秒以内に応答する
- **SC-003**: タイムスタンプメモ一覧を持つ動画詳細画面が3秒以内にロードされる（メモが50件以内の通常ケース）
- **SC-004**: 神回・お気に入り一覧画面でお気に入りのタイムスタンプが正しく一覧表示され、推し別・タグ別・年月別フィルターが機能する
- **SC-005**: 他ユーザーのタイムスタンプメモ・動画ノートへの参照・更新・削除操作がすべて拒否される（権限テスト100%通過）
- **SC-006**: メモ本文のHTMLタグが画面上で意図せず描画されることなく、テキストとして表示される（XSS対策100%通過）
- **SC-007**: 非公開・削除済み動画のメモ一覧ページが正常に表示され「この動画は現在YouTubeで再生できません」等の通知が表示される

## Clarifications

### Session 2026-06-20

- Q: 動画ノートの本文が空になった時の挙動 → A: 本文が空のとき保存ボタンを無効化する。削除は明示的な「削除」ボタンで行う（Option B）
- Q: ユーザー固有タグの作成タイミング → A: タイムスタンプメモの入力フォーム内でタグ名をそのまま入力してインライン作成できる（Option A）
- Q: 神回・お気に入り一覧の「お気に入り動画」タブのスコープ → A: タイムスタンプメモのお気に入りのみ。動画お気に入りは将来拡張とする（Option B）
- Q: 動画全体ノートの保存UI方式 → A: 明示的な「保存」ボタンで手動保存し、完了フィードバックを表示する（Option A）
- Q: タイムスタンプメモ保存後のリスト更新方式 → A: サーバー保存成功後のレスポンスを受け取ってから Alpine.js でリストに追加する（Option A）

## Assumptions

- Feature 5（player-and-playback）が完了しており、`user_watch_items`・YouTube IFrame Player連携・動画詳細画面が実装済みであることを前提とする
- タイムスタンプメモ一覧は秒数昇順で表示する（デフォルトソート）
- ユーザー固有タグのスラッグはシステムが自動生成する（名前からkebab-case変換等）
- モバイル表示では動画プレイヤーの下部にメモ欄を配置する（CLAUDE.md Design Guidelinesに準拠）
- タイムスタンプの表示形式はMM:SS（60分以上の動画はHH:MM:SS）とする
- 神回・お気に入り一覧はタイムスタンプメモのお気に入り（`timestamp_memos.is_favorite`）のみを対象とする。お気に入り動画（`user_watch_items.is_favorite`）は将来拡張とし、本Featureでは `user_watch_items` テーブルを変更しない
- AIによるタグ付け自動提案は将来拡張とし、本Featureのスコープ外とする
- タグの検索・フィルタリング（メモ一覧でのタグ絞り込み）はMVPで実装する。タグ管理専用画面（タグの一覧・削除）は必要に応じて追加する
- メモ本文はプレーンテキストとして保存・表示する。Markdown記法等のリッチテキストは将来拡張とする
