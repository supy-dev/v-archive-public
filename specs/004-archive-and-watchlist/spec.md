# Feature Specification: アーカイブ閲覧と見るリスト管理

**Feature Branch**: `004-archive-and-watchlist`

**Created**: 2026-06-20

**Status**: Draft

**Input**: User description: "feature 4の仕様を作成してください！"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 新着アーカイブを閲覧して見るリストに追加する (Priority: P1)

登録チャンネルの動画が同期されると、ユーザーは「新着アーカイブ」画面でまだ何もしていない動画（未整理）をリスト表示できる。各動画に対して「見るリストに追加」または「見送る」を選択できる。

**Why this priority**: 動画同期（Feature 003）で取得されたデータをユーザーが最初に活用する接点であり、本サービスの中心的な価値を生み出す最重要フロー。

**Independent Test**: 同期済みチャンネルがある状態で新着アーカイブ画面を開き、動画1件に「見るリストに追加」を押すと `user_watch_items` に `want_to_watch` ステータスで追加されること、「見送る」を押すと `skipped` ステータスで作成されること。

**Acceptance Scenarios**:

1. **Given** 登録チャンネルに同期済みの動画が存在し、ユーザーはその動画を一度も操作していない, **When** 新着アーカイブ画面を開く, **Then** 未整理（`user_watch_items` が存在しない）の動画が新着順に表示される
2. **Given** 未整理の動画カードが表示されている, **When** 「見るリストに追加」ボタンを押す, **Then** `user_watch_items` が `want_to_watch` で作成され、その動画がアーカイブ一覧から消える
3. **Given** 未整理の動画カードが表示されている, **When** 「見送る」ボタンを押す, **Then** `user_watch_items` が `skipped` で作成され、動画がアーカイブ一覧から消える
4. **Given** アーカイブ一覧に複数推しの動画が混在している, **When** 特定の推しでフィルタリングする, **Then** 選択した推しのチャンネルの動画のみ表示される
5. **Given** 利用可能でない（`is_available=false`）の動画, **When** 新着アーカイブ一覧を閲覧する, **Then** その動画は一覧から完全に除外されて表示されない
6. **Given** 未整理の動画がない, **When** 新着アーカイブ画面を開く, **Then** 「新着の未整理動画はありません」などの空状態メッセージが表示される

---

### User Story 2 - 見るリストの視聴ステータスを管理する (Priority: P1)

ユーザーは「見るリスト」画面でステータス別（未視聴・視聴中・視聴済み・見送り）にタブ切替できる。各動画の視聴ステータスを手動で変更できる。

**Why this priority**: US1 で追加した動画を管理する場所であり、サービスの継続利用を支えるコア画面。

**Independent Test**: `want_to_watch` の `user_watch_items` がある状態で見るリスト「未視聴」タブを開くと動画が表示され、「視聴済み」に変更すると `watched` に更新されて「視聴済み」タブへ移動すること。

**Acceptance Scenarios**:

1. **Given** `user_watch_items` に複数ステータスの動画がある, **When** 見るリスト画面の「未視聴」タブを表示する, **Then** `want_to_watch` の動画のみ表示される
2. **Given** 見るリストの動画カードが表示されている, **When** ステータスを「視聴済み」に変更する, **Then** `status` が `watched`、`watched_at` が現在時刻に更新され、「未視聴」タブから消えて「視聴済み」タブへ移る
3. **Given** 見るリストの動画カードが表示されている, **When** ステータスを「見送る」に変更する, **Then** `status` が `skipped`、`skipped_at` が更新される
4. **Given** 見るリストの「視聴済み」タブを表示している, **When** ステータスを「未視聴」に戻す, **Then** `status` が `want_to_watch` に更新され、「視聴済み」タブから消える
5. **Given** 別ユーザーの `user_watch_items` が存在する, **When** 自分の見るリスト画面でステータス変更を試みる, **Then** 他ユーザーのアイテムは表示されず操作もできない

---

### User Story 3 - ホーム画面で視聴状況のサマリーを確認する (Priority: P2)

ホーム画面に「未整理件数」「見るリスト件数」「視聴中件数」「視聴済み件数」のサマリーを表示し、動線ボタンから各画面へ遷移できる。

**Why this priority**: US1/US2 が完成して初めてデータが溜まるため P2。サービスのダッシュボードとしてリテンションに貢献するが US1/US2 なしでは意味がない。

**Independent Test**: `user_watch_items` が混在する状態でホーム画面を開くと、正しい件数がサマリーカードに表示されること。

**Acceptance Scenarios**:

1. **Given** ユーザーに未整理動画10件、見るリスト5件、視聴中2件、視聴済み20件がある, **When** ホーム画面を開く, **Then** 各件数が正確なカウントで表示される
2. **Given** ホーム画面のサマリーカード「新着」をクリックする, **When** 遷移する, **Then** 新着アーカイブ画面（未整理フィルタ済み）へ移動する
3. **Given** ホーム画面のサマリーカード「見るリスト」をクリックする, **When** 遷移する, **Then** 見るリスト画面へ移動する

---

### Edge Cases

- 同じ動画に対して同一ユーザーが重複して `user_watch_items` を作成しようとした場合はエラーなく upsert またはバリデーションで防ぐ
- 登録チャンネルが一件もない場合、新着アーカイブは空状態を表示する
- `youtube_videos` にはあるが `user_channels` 経由で自分の推しに紐づかない動画は表示しない（他ユーザーが登録したチャンネルの動画を見せない）
- `priority`（優先度）はデフォルト 0 として MVP では変更 UI を省略する
- ページネーションまたは無限スクロールを用いてデータ量増大に対応する（件数上限なしの全件表示は禁止）
- ステータス変更は楽観的 UI（即座に表示変更）とするかサーバー応答後更新かは実装者判断に委ねる

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは自分の登録チャンネルに紐づく `youtube_videos` の一覧を `published_at` 降順（YouTube公開日の新しい順）で閲覧できなければならない
- **FR-002**: 新着アーカイブ一覧は、`user_watch_items` が存在しない動画（未整理）を初期状態で表示しなければならない
- **FR-003**: ユーザーは動画に対して「見るリストに追加（`want_to_watch`）」または「見送る（`skipped`）」を選択できなければならない
- **FR-004**: 新着アーカイブ一覧は、推し・動画種別（アーカイブ/ライブ/ショート等）・未整理のみ、でフィルタリングできなければならない
- **FR-005**: ユーザーは見るリスト画面から `want_to_watch`（未視聴）・`watched`（視聴済み）・`skipped`（見送り）の3状態を手動変更できなければならない。`watching`（視聴中）への変更は Feature 005 のプレイヤーから自動設定され、このFeatureでは手動変更UIを提供しない
- **FR-006**: 見るリスト画面はステータス別タブで動画を表示し、各タブは対応するステータスの動画のみを表示しなければならない
- **FR-007**: 同一ユーザーが同一動画に対して `user_watch_items` を重複作成しないこと（`(user_id, youtube_video_id)` 複合ユニーク制約）
- **FR-008**: ステータス変更時に対応するタイムスタンプ（`started_at`・`watched_at`・`skipped_at`）を自動設定しなければならない
- **FR-009**: ユーザーは自身の `user_watch_items` のみ閲覧・更新でき、他ユーザーのデータは操作できないこと（所有権の強制）
- **FR-010**: ホーム画面に未整理件数（全登録チャンネル・全期間の `user_watch_items` 未作成動画数）・見るリスト件数（`want_to_watch`）・視聴中件数（`watching`）・視聴済み件数（`watched`）のサマリーを表示しなければならない
- **FR-011**: `is_available=false` の動画は新着アーカイブ一覧から完全に除外し、一覧クエリで `is_available=true` の動画のみを対象とすること
- **FR-012**: 一覧表示は必ずページネーションまたはカーソルベースの範囲取得を用い、全件一括取得を避けなければならない
- **FR-013**: 新着アーカイブ一覧と見るリスト一覧での動画表示に、サムネイル・タイトル・推し名・公開日時・動画時間を含めなければならない
- **FR-014**: `status` は列挙値として保存し、任意の文字列を保存してはならない
- **FR-015**: ユーザーは見るリストから `user_watch_items` を完全削除できなければならない。削除後、その動画は「未整理」状態に戻り新着アーカイブ一覧に再表示される。削除権限は所有者のみ（FR-009 に準ずる）

### Key Entities *(include if feature involves data)*

- **UserWatchItem**: ユーザーが操作した動画ごとに1件作成されるユーザー固有の視聴アイテム。`status`（視聴ステータス）、`priority`（優先度）、`is_favorite`（お気に入り）、各タイムスタンプ（`added_at`・`started_at`・`watched_at`・`skipped_at`）、`last_position_seconds`（最終再生位置）を保持する
- **WatchStatus**: `want_to_watch`（未視聴）・`watching`（視聴中）・`watched`（視聴済み）・`skipped`（見送り）の4値。列挙型として扱う
- **YoutubeVideo（既存）**: 共有マスタ。このFeatureではデータの更新は行わず閲覧のみに使用する
- **YoutubeChannel（既存）**: チャンネルと推し（Oshi）の紐づきを経由して、ユーザーが閲覧対象とする動画の絞り込みに使用する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが新着アーカイブ一覧を開いてから「見るリストに追加」まで3タップ以内で完了できる
- **SC-002**: 一覧画面の初期表示が、登録チャンネルに100件以上の動画があっても3秒以内に完了する
- **SC-003**: 視聴ステータスの変更操作が確定するまで2秒以内に応答が返る
- **SC-004**: 同一動画への重複 `user_watch_items` 作成が0件であること（データ整合性）
- **SC-005**: 他ユーザーのデータを閲覧・変更するリクエストが100%拒否されること（所有権保護）
- **SC-006**: ホーム画面のサマリー件数が実際のデータと一致すること（正確性）

## Clarifications

### Session 2026-06-20

- Q: Feature 004 の見るリストで `watching`（視聴中）への手動変更が必要か？ → A: 不要。`watching` は Feature 005 のプレイヤーから自動設定のみ。Feature 004 では `want_to_watch` / `watched` / `skipped` の3状態のみ手動変更対象。
- Q: `is_available=false` の動画を新着アーカイブ一覧でどう扱うか？ → A: 完全に除外（一覧クエリで `is_available=true` 条件を付け、非表示にする）。
- Q: 見るリストから `user_watch_items` を完全削除できるか？ → A: できる。削除すると「未整理」状態に戻り新着アーカイブ一覧に再表示される。
- Q: 新着アーカイブ一覧の「新着順」のソート基準は？ → A: `youtube_videos.published_at` 降順（YouTube公開日の新しい順）。
- Q: ホーム画面「未整理件数」の算出範囲は？ → A: 全登録チャンネルの全期間を対象（期間絞り込みなし）。

## Assumptions

- Feature 001（認証基盤）・Feature 002（推し・チャンネル管理）・Feature 003（動画同期）が実装済みで、`youtube_videos` にデータが存在する前提で動作する
- `last_position_seconds`（最終再生位置）の保存はこのFeatureでは対象外とし、Feature 005（プレイヤー・再生連携）で実装する
- お気に入り（`is_favorite`）登録はテーブルにカラムとして持つが、UI操作はこのFeatureの範囲外とする（Feature 006以降）
- `priority`（優先度）の変更UIはMVPでは省略し、デフォルト0のままとする
- タイムライン/カレンダー表示は対象外。シンプルなリスト・カード表示のみ
- 動画種別フィルタはチェックボックスまたはドロップダウン形式とする（具体的なUI形式は実装者判断）
- `want_to_watch` のみを「見るリスト件数」として見るリスト画面の初期タブに表示する（`watching` と `watched` は別タブ）
- ページネーションは1ページあたり20〜30件を基本とする
